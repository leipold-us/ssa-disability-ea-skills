# Contributing

Contributions are welcome. This project covers a complex and evolving domain —
SSA policy changes, new TEFCA developments, and market shifts all create
opportunities to keep these skills current and accurate.

## What to contribute

**High priority:**
- POMS chapter coverage not yet included (DI 110 Initial Claims, DI 120 Appeals,
  DI 280 Continuing Disability Reviews, DI 900 Drug Addiction and Alcoholism)
- Updates as SSA's TEFCA integration matures beyond spring 2026 go-live
- Hearing level workflow detail (HALLEX — Hearings, Appeals, and Litigation Law Manual)
- State DDS operational variations that affect integration

**Always welcome:**
- Corrections to any policy interpretation — POMS is the authoritative source
- New primary sources (OIG reports, SSA announcements, RFI responses)
- Named contacts and organizational updates
- Market developments (new HIT partners, network changes, vendor landscape)

## How to contribute

1. Open an issue first for significant additions or corrections
2. Fork the repository
3. Make changes with citations to primary sources
4. Submit a pull request with a brief description of what changed and why

## Citation standards

Every factual claim in these skills should be traceable to a primary source:
- POMS sections (cite section number and effective date)
- SSA publications (cite URL and access date)
- OIG reports (cite report number and date)
- News/analysis (cite publication, author, and date)

Do not add content based on secondary sources alone if a primary source exists.
SSA POMS is publicly accessible at https://secure.ssa.gov/apps10/poms.nsf/

## Policy interpretation

These skills make architectural inferences from policy — for example, concluding
that medical equivalence requires human MC/PC review (DI 24508.010) means AI
cannot make a final Step 3 equivalence finding. If you believe an interpretation
is incorrect, open an issue with the relevant POMS citation.

## Accuracy over completeness

A smaller, accurate skill is more valuable than a larger, partially incorrect one.
When in doubt, open an issue rather than adding uncertain content.
